# 学び: Gray-Scott は「種(V)」を殺すと二度と模様が出ない

記録日: 2026-06-25 / 関連 Issue: #11（発見）, #12, #13（修正）

## 何が起きたか

Phase 2 を実機で開いたら、起動後ずっと **画面が真っ黒**で核生成も成長も起きなかった。
パネルの「経過%」は増えていた＝ループは正常に回っていたので、初期化失敗ではなく**パラメータ設計のバグ**だった。

## なぜ真っ黒になったのか

Gray-Scott 反応拡散は、状態に U（基質）と V（生成物）の 2 成分を持つ。
反応式の心臓部はこれ:

```
db/dt = Db·∇²V + U·V²  − (kill + feed)·V
                ^^^^^^
                反応項
```

ポイントは **反応項が `U·V²`** であること。V が掛かっているので、
**V が 0 の場所では反応が一切起きない**。つまり:

> 一度 V が画面全体で 0 になると、`U·V² = 0` なので V は二度と生まれない。
> 種が死んだら終わり。

旧コードの静穏状態 `QUIESCENT` は `feed=0.0545 / kill=0.07` だった。
この値は Gray-Scott のパラメータ空間の中でも **「V が消滅する領域」** に当たる。
`seed.ts` が中央に置いた V の種は、数秒で全面 0 に減衰して死ぬ。
その後いくら成長パラメータ `GROWTH`(kill=0.062) に遷移しても、**育てる種がもう無い** → 永遠に黒。

数値で言うと、kill が「育つ域」(<0.065) に入るのは約 22 秒後。それまでに種は死んでいた。

## なぜ「dt レバー」だと直るのか

修正では、冷却の表現を **kill から dt（反応の時間刻み）** へ移した。

- `QUIESCENT = { ...GROWTH, dt: 0.15 }`
  → feed/kill は GROWTH と同じ「育つ域」のまま固定し、**dt だけ小さく**する
- dt は「1 ステップでどれだけ状態を進めるか」。小さいと状態がほとんど変化しない
  ＝反応がほぼ止まって見える（＝静穏）が、**V は減衰せず生き残る**
- 時間が進むと dt が `0.15 → 1.0` へ上がり、同じ種から核生成→成長が始まる

```
旧: 冷却 = kill を上げる → V が消滅領域へ → 種が死ぬ → 真っ黒
新: 冷却 = dt を上げる   → kill は育つ域で固定 → 種は生存 → 反応が加速して成長
```

「冷えると反応が速くなる」という物語にもなって、むしろ自然になった。

## 一般化した教訓

1. **反応項に変数が掛かる系（`U·V²` 等）では、その変数を 0 にしてはいけない。**
   0 は吸収状態（absorbing state）で、そこから戻れない。
2. **「模様を止めたい」とき、反応を殺す（致死域に入れる）のではなく、時間を遅くする。**
   速度レバー（dt）と生死レバー（kill）を混同しない。
3. **パラメータを時間で動かすときは「全工程で致死域を跨がない」ことをテストで保証する。**
   → `genesis.test.ts` に「全 t で kill が致死域に入らない／dt が単調増加」を追加済み。
